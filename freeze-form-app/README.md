# Freeze Form Generator

Веб-приложение для создания документов «Official Freeze Form» по шаблону.
Заполняешь форму — документ сразу отображается на странице. Даты считаются
автоматически. Скачать `.docx` / `PDF` можно по необходимости (для подписи).

## Файлы

| Файл | Назначение |
|------|------------|
| `app.py` | Веб-приложение (Streamlit) |
| `freeze_core.py` | Логика: расчёты, рендер, конвертация PDF |
| `freeze_template_jinja.docx` | Шаблон документа с метками `{{ ... }}` |
| `requirements.txt` | Python-зависимости |
| `packages.txt` | Системные пакеты для хостинга (LibreOffice → PDF) |

## Запуск локально

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется в браузере на `http://localhost:8501`.
Для PDF нужен установленный [LibreOffice](https://www.libreoffice.org/download).
Без него работает всё, кроме кнопки PDF.

## Деплой (публичная ссылка) — Streamlit Community Cloud, бесплатно

1. Залей все эти файлы в репозиторий на GitHub.
2. Зайди на **share.streamlit.io**, войди через GitHub.
3. **Create app** → выбери репозиторий, ветку и `app.py` как главный файл → **Deploy**.
4. `requirements.txt` и `packages.txt` подтянутся автоматически
   (LibreOffice ставится из `packages.txt`, поэтому PDF работает и в облаке).
5. Получишь публичный URL вида `https://<имя>.streamlit.app` — её и отправляешь.

> Первая генерация PDF после старта может занять несколько секунд
> (LibreOffice прогревается). Альтернатива с тем же набором файлов —
> Hugging Face Spaces (тип Space: Streamlit).

## Редактирование шаблона

Шаблон — обычный Word-файл с метками `{{ ... }}`.

- В приложении: сайдбар → **Скачать шаблон**, открой в Word, поменяй текст и
  форматирование (метки оставь на месте), затем **Загрузить свой шаблон**.
- Чтобы изменения стали постоянными для всех — замени `freeze_template_jinja.docx`
  в репозитории на свою версию.

Доступные метки (их значения подставляет приложение):

| Метка | Значение |
|-------|----------|
| `{{ exhibit }}` | Буква доп. соглашения (A, B, C…) |
| `{{ client_name }}` | Имя клиента |
| `{{ selected_plan }}` | План клиента |
| `{{ start_date }}` | Дата создания Хаба |
| `{{ orig_expiration }}` | Изначальная дата окончания |
| `{{ break_start }}` | Дата старта паузы |
| `{{ break_end }}` | Дата окончания паузы (расчёт) |
| `{{ break_days }}` | Длительность паузы в днях |
| `{{ reason }}` | Причина (или N/A) |
| `{{ adjusted_expiration }}` | Новая дата окончания (расчёт) |

**Важно:** менять текст и формат вокруг меток — можно сколько угодно. Если нужно
добавить *новое поле* (новую метку), его придётся также добавить в форму в `app.py`.

## Логика расчётов

- Длительность паузы и дату окончания можно задать любую из двух — вторая считается.
- Новая дата окончания контракта = изначальная дата окончания + дни паузы.
- Пустая причина → подставляется `N/A`.
- Поля подписей/дат/паспорта в документе остаются пустыми (под ручное подписание).
